# AI Medical Diagnostic Assistant (AMDA)

**Версия:** 1.2
**Дата:** 11 апреля 2026
**Язык:** русский (по умолчанию)

AMDA — специализированная диагностическая система на базе Claude Code, реализующая строгий последовательный протокол дифференциальной диагностики с сохранением состояния между сессиями.

> **Важно:** Система является вспомогательным инструментом. Окончательный диагноз и лечение может поставить только лицензированный врач. Не используйте полученные рекомендации без очной консультации специалиста.

---

## Возможности

- **Структурированный сбор анамнеза** — жалобы, история болезни, жизненный анамнез, вредные привычки
- **Анализ медицинских документов с OCR** — поддержка PDF, JPG, PNG, DOCX с автоматическим распознаванием текста
- **Дифференциальная диагностика** — байесовский подход, список диагнозов с вероятностями (сумма = 100%)
- **Приоритизация исследований** — минимально необходимый набор тестов для максимальной диагностической ценности
- **Детектирование красных флагов** — автоматическое выявление критических симптомов и рекомендации по экстренному обращению
- **Stateful-режим** — все данные пациента сохраняются в файлах проекта и доступны между сессиями
- **Финальный диагноз** — формируется только при достижении вероятности ≥ 90%

---

## Диагностический цикл

```
1. Сбор и структуризация анамнеза
        ↓
2. Анализ медицинских документов (если загружены)
        ↓
3. Формирование дифференциального диагноза
        ↓
4. Приоритетный план дополнительных исследований
        ↓
5. Итеративное уточнение (при получении новых данных)
        ↓
6. [Повторяем 3–5, пока хотя бы один диагноз не достигнет ≥ 90%]
        ↓
7. Финальный диагноз и рекомендации
```

## Быстрый старт

1. Откройте проект в Claude Code:
   ```bash
   cd medical_diagnostic_assistant
   claude
   ```

2. Система автоматически приветствует пациента и начинает сбор анамнеза.

3. Для загрузки медицинских документов (анализы, снимки, выписки) поместите файлы в папку `documents/` и сообщите об этом системе.

4. Следуйте инструкциям AMDA — система проведёт через все этапы диагностики.

---

## Управление данными пациента

- Все данные хранятся **локально** в папке `patient-data/` — ничего не отправляется на внешние серверы.
- Для анонимности рекомендуется использовать ID вместо ФИО (например, `Пациент_2026_001`).
- При завершении диагностики кейс автоматически переносится в `patient-data/archive/`.
- Для удаления данных пациента удалите соответствующие файлы из `patient-data/`.

- Модель: `claude-sonnet-4-6` или выше (рекомендуется `claude-opus-4-6` для лучшей точности диагностики)

---

## Ограничения и отказ от ответственности

- AMDA **не заменяет** лицензированного врача и не является медицинским прибором.
- Система не назначает конкретные препараты или дозировки до подтверждённого диагноза ≥ 90% и без рекомендации врача.
- При наличии критических симптомов (красные флаги) система настоятельно рекомендует немедленно обратиться за медицинской помощью.
- Клинические рекомендации в системе ориентированы на стандарты 2025–2026 гг. и требуют проверки актуальности при использовании.
